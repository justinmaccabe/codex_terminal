Methodology Specification

codex_terminal

Initial universe, signal framework, macro rules, benchmark logic, and input schemas for v1 implementation.

1. Purpose

This document freezes the first-pass methodology for codex_terminal v1. It is designed to be good enough to build against without pretending that every research choice is permanent. The aim is disciplined first implementation, not premature perfection.

2. Methodology Principles

  • Prefer simplicity, transparency, and stability over a fragile “optimal” model.
  • Use liquid ETF proxies from approved providers only.
  • Separate structural and tactical signals.
  • Use macro regime as context and mild adjustment, not a black-box controller.
  • Let the portfolio lab be stress-aware, not only variance-aware.
  • Expose uncertainty when a proxy or data source is weak.

4. Sleeve Classification Framework

Each instrument in the universe should be tagged with:

  • asset class,
  • region,
  • style or factor exposure,
  • inflation sensitivity,
  • duration sensitivity,
  • risk-on or diversifier classification,
  • strategic-core eligibility,
  • tactical-overlay eligibility.

5.3 Macro Signal

Macro regime fit should be a sleeve-level compatibility score based on the current FRED-derived quadrant. This should mildly adjust conviction rather than dominate the ranking system.

5.4 Standardization Rule

All continuous screener signals should be normalized into comparable ranks or z-scores before being blended. V1 should avoid exotic transformations.

5.5 Composite Score Rule

The initial v1 composite should use block weights rather than arbitrary per-signal hand tuning.

  • Tactical block: 60 percent of total composite
  • Structural block: 30 percent of total composite
  • Macro block: 10 percent of total composite

Within each block, signals are equal-weighted if available. If a signal is unavailable for a sleeve, redistribute weight evenly across available signals within that block.

Watchlist is an override label for assets with rapidly improving tactical signals but insufficient composite support for Overweight.

6. Macro Regime Methodology

6.1 Framework

Use a four-quadrant rules-based model:

  • Growth up / inflation up
  • Growth up / inflation down
  • Growth down / inflation up
  • Growth down / inflation down

6.2 V1 Inputs

Macro Concept Default FRED Directional Proxy Purpose
GrowthIndustrial Production and unemployment trend proxyClassify growth direction
InflationCPI YoY and inflation trend proxyClassify inflation direction
Policy / Rates ContextFed Funds and Treasury curve slopeContext only in v1
Liquidity Stress ContextCredit spread or stress proxy if available on FREDNarrative context and stress flag

6.3 V1 Classification Rule

  • Growth direction is based on whether the selected growth composite is improving or deteriorating versus a trailing baseline.
  • Inflation direction is based on whether inflation is accelerating or decelerating versus a trailing baseline.
  • The regime is assigned from the sign combination of those two directions.
  • Confidence can be reduced when the growth and inflation signals are weak or conflicting.

6.4 Regime-to-Asset Mapping

V1 should maintain a qualitative mapping table of which sleeves tend to benefit or struggle in each regime. This table is explanatory, not a hard allocation engine.

7. Portfolio Lab Methodology

7.1 Portfolio Architecture

  • Strategic core: diversified long-run exposures across equities, duration, inflation protection, real assets, and diversifiers.
  • Tactical overlay: tilts based on the screener composite and risk context.
  • Risk target: annualized realized volatility aligned to SPY.

7.2 Initial Optimization Modes

Mode Primary Objective Notes
Mean-VarianceMaximize expected return per unit of varianceMust be constrained and presented as one lens, not the truth
Target VolatilityScale portfolio toward SPY volatilityDefault practical mode
Target Risk ContributionBalance risk across sleevesUseful diversification lens
Stress-Aware ScorePrefer portfolios with better downturn resilienceUse as ranking overlay, not a separate classical optimizer

7.3 Stress-Aware Score Components

  • Maximum drawdown
  • Downside deviation
  • Sortino ratio
  • Crisis-period return differentials
  • Correlation behavior in selloffs
  • Inflation-shock and growth-shock period behavior

7.4 Leverage Methodology

  • Compute base diversified portfolio realized volatility.
  • Compute trailing realized volatility of SPY.
  • Scale the portfolio toward SPY vol using a capped leverage multiplier.
  • Apply a user-visible financing-cost assumption.
  • Show both gross and net-of-financing results where possible.

7.5 Rebalancing Methodology

  • Default trigger: weight drift beyond threshold.
  • Optional trigger: realized volatility departure beyond threshold.
  • Calendar reporting can exist, but calendar rebalancing is not the house default.

7.6 Concentration Policy

Use soft concentration controls. V1 should include warning thresholds but not hard stop constraints. Suggested starting thresholds:

  • single sleeve warning above 25 percent,
  • equity aggregate warning above 65 percent,
  • any one non-equity diversifier warning above 20 percent.

8. Benchmark Methodology

8.1 SPY

SPY is the permanent public-market anchor benchmark and volatility target reference.

8.2 Market Beating Portfolio

This is the internally constructed house benchmark generated by the Portfolio Lab methodology.

8.3 Vanguard Target Retirement

  1. Attempt automated history/NAV retrieval.
  2. If successful, include directly as a benchmark series.
  3. If not successful, mark benchmark status as degraded.
  4. Prompt user for latest close/NAV input.
  5. Use partial comparison where history is missing, and state that clearly.

9.3 Validation Rules

  • Normalize weights if the file appears to use percentages.
  • Reject blank ticker rows.
  • Flag duplicate tickers for consolidation or review.
  • Warn if total weight differs materially from 100 percent.
  • Flag unsupported tickers and continue with partial analysis where feasible.

10. Exposure Inference Methodology

For uploaded portfolios, v1 should use a practical mapping-first approach:

  • If the holding is in the known ETF universe, use direct sleeve classification.
  • If the holding is an ETF outside the house universe, classify using available fund metadata and broad proxy mapping where possible.
  • If the holding is an individual stock or unsupported instrument, classify as unresolved or broad equity fallback if that is the best honest approximation.
  • Display unresolved exposures explicitly rather than burying them.

11. Factor Attribution Methodology

V1 should support regression-based factor attribution at a practical level. The first pass does not need a fully custom research library.

  • Use available factor proxies or factor datasets where feasible.
  • Support at least market, size, value, and momentum attribution in some form.
  • Treat factor attribution as an explanatory layer rather than a trading engine.

12. AQR Expected Return Context

AQR capital market expectations should be displayed as contextual reference only. V1 methodology rules:

  • do not use AQR expectations directly in optimizer objective functions,
  • do not use them as a direct tactical timing signal,
  • use them to frame whether the house portfolio’s exposures align with or diverge from the medium-term outlook.

13. Missing Data Rules

  • If a ticker cannot be fetched, show it as unavailable and exclude it from calculations that require a full series.
  • If a signal cannot be built credibly, mark it unavailable rather than substitute noise.
  • If a benchmark is partially available, clearly distinguish between full-history and current-point-only comparisons.
  • Every degraded-data state should be visible in the UI.

14. What Is Frozen vs Flexible

14.1 Frozen For V1

  • module structure,
  • approved provider policy,
  • starter ETF universe, subject only to the managed-futures exception noted above,
  • three-block screener design,
  • four-quadrant macro framework,
  • SPY-anchored risk target,
  • stress-aware portfolio evaluation.

14.2 Flexible After V1

  • exact factor data source,
  • additional sleeves and proxies,
  • signal weighting refinements,
  • regime-threshold refinements,
  • more advanced carry and valuation models.
